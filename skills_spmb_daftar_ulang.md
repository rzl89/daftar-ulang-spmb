# Skills for SPMB Re-Registration Web

## Project Goal
Build a professional, clean, trustworthy, and easy-to-use re-registration website for a vocational high school. The interface should feel official, well-structured, and suitable for school administration, students, and parents/guardians.

## Core Principles
- Prioritize clarity, simplicity, and trust.
- Keep the interface formal, neat, and easy to understand.
- Avoid decorative elements that reduce readability.
- Make the most important actions obvious.
- Design for real users who may not be technically skilled.
- Use a mobile-first approach because many users will access the site from phones.

## Visual Style
- Use a clean, modern, and institutional look.
- Follow the school's branding identity when available, including logo and official colors.
- Keep the color palette minimal and consistent.
- Use enough whitespace to make the layout feel organized.
- Avoid excessive shadows, gradients, animations, or noisy backgrounds.
- Use rounded corners sparingly and consistently.
- Make sections visually separated but not crowded.

## Typography
- Use a professional sans-serif typeface.
- Maintain clear hierarchy for headings, subheadings, labels, and body text.
- Keep font sizes readable on both desktop and mobile.
- Use consistent spacing between titles, text, and form elements.
- Avoid using too many font weights.

## Layout and Structure
- Use a clear page hierarchy: header, main content, supporting information, and footer.
- Organize content into logical sections.
- Keep forms and instructions separated so users can scan the page easily.
- Use cards, panels, or section blocks only when they improve readability.
- Prevent content from feeling stacked without structure.
- Keep the main workflow visible and easy to follow.

## User Experience Rules
- Make the registration flow simple and linear.
- Show users exactly what to do next.
- Use short, clear instructions.
- Reduce the number of clicks and steps whenever possible.
- Avoid confusing terminology; use language familiar to students and parents.
- Provide helpful feedback after every important action.
- Make success, warning, and error messages easy to understand.

## Forms and Validation
- Use clean, organized form layouts.
- Group related fields together.
- Mark required fields clearly.
- Use appropriate input types for each field.
- Validate data before submission.
- Display validation messages directly near the field.
- Use clear error messages that explain how to fix the issue.
- Preserve user input when validation fails.
- Prevent accidental duplicate submissions.

## Data and Content Rules
- Present student data, payment data, and registration status in a structured format.
- Use tables or summary cards only when they improve readability.
- Keep labels consistent across the application.
- Use official school terminology where appropriate.
- Make status indicators clear, such as:
  - Pending
  - Verified
  - Approved
  - Rejected
  - Completed
- Ensure all displayed information is easy to scan and verify.

## Component Standards
- Build reusable components for buttons, inputs, cards, tables, alerts, badges, and modals.
- Keep components small and focused on a single purpose.
- Avoid duplicated UI logic.
- Use consistent spacing, border radius, and interaction states.
- Keep component names descriptive and meaningful.
- Ensure the same component behavior across all pages.

## Navigation
- Keep the navigation simple and obvious.
- Highlight the current page clearly.
- Do not overload the user with too many menu items.
- Use clear labels such as Home, Registration, Status, Upload Documents, Instructions, and Contact.
- Make navigation usable on both desktop and mobile.

## Accessibility
- Maintain sufficient contrast between text and background.
- Ensure all interactive elements are easy to click or tap.
- Support keyboard navigation.
- Provide labels for form fields.
- Use semantic HTML wherever possible.
- Do not rely on color alone to communicate status.

## Responsiveness
- The interface must work well on mobile, tablet, and desktop.
- Prevent horizontal scrolling unless absolutely necessary.
- Ensure forms and tables adapt gracefully to small screens.
- Stack elements properly on mobile.
- Keep buttons large enough to tap comfortably.

## Performance
- Keep the website fast and lightweight.
- Avoid unnecessary libraries or heavy animations.
- Optimize images and assets.
- Load only what is needed for the current page.
- Keep interactions smooth even on low-end devices.

## Security and Reliability
- Treat user data carefully.
- Validate input on both client and server when possible.
- Protect sensitive student information.
- Avoid exposing internal system details in error messages.
- Provide safe and clear session behavior.
- Prevent data loss during form filling.

## Error Handling
- Use friendly and specific error messages.
- Explain what happened and what the user should do next.
- Handle loading, empty, and failure states gracefully.
- Never show broken layouts or unclear system states.
- Provide retry options when appropriate.

## Content Tone
- Use polite, formal, and reassuring language.
- Keep instructions simple and direct.
- Avoid slang, jokes, or overly casual wording.
- Use language suitable for school administration.
- Write messages that help users feel confident and guided.

## Page-Specific Expectations
### Home Page
- Explain the purpose of the website clearly.
- Show a short guide to the registration process.
- Provide quick access to the main action.

### Registration Page
- Keep the form clean and well organized.
- Show progress clearly if registration has multiple steps.
- Make document upload instructions easy to understand.

### Status Page
- Show registration progress in a clear and structured way.
- Use badges or steps to show current status.
- Make verification results easy to read.

### Document Upload Page
- Explain file requirements clearly.
- Show supported file types and size limits.
- Confirm uploads successfully with clear feedback.

### Admin or Verification Area
- Make data review fast and structured.
- Allow easy filtering, sorting, and searching.
- Highlight important alerts and pending items.

## Output Quality Rules
- Always produce polished, production-ready UI.
- Keep the design consistent across all pages.
- Avoid random styling changes from page to page.
- Ensure the final result looks like a real school system, not a demo.
- Favor reliability, clarity, and structure over flashy visuals.

## Final Rule
Always prioritize a professional school administration experience that is neat, structured, trustworthy, and easy to use for students, parents, and staff.
